Week 03 - Creating the web page

Before the Lesson Begins

  • This week the students will write about 90% of the HTML for the home page for the rafting site. 

During the Lesson

  • You can have them practice with new CSS concepts this week by doing something together in class. Here is a continuation of last week's page that you can use to practice Classes and IDs
  • The Site plan template contains a wireframe that we will be using this week.  Stress two things to the students: first, the assignment will make much more sense if they refer to the wireframe often while reading the instructions, and second their page will NOT look like the wireframe at the end of this week. We will have all of the content represented there on our page, but it will be weeks before our page looks just like the wireframe. 
  • Practice with the student one day in class on how to take a wireframe and make HTML code using the wireframe. I would go over all 5 steps in the W03 - Working with Wireframes activity. You will be using the wireframe from dribbble.com Here's a follow-up video to the one they watch in preparation explaining what you might do in class after studying the groupings of a wireframe:
  • The 'Troubleshooting with Validation' is a good exercise to do during class as well so the students can help each other or you can do it together.
  • The Web Frontend Developer information is something they can do on their own. You can require a report from your students if you want.
  • Make sure to stress the importance of completing the Troubleshooting and Wireframe activities as well. They are ungraded, but important that they understand.
  • For some students using Mac computers, the hero.png images causes issues because of its size. They will get a 400 error when they go to push that image. (...RPC failed; HTTP 400 curl 22...) If that happens run this command on their terminal to avoid it in the future: git config --global http.postBuffer 157286400
  • Help the students understand constructive versus destructive criticism. The criticism needs to be valid and concrete
  • Make sure you are available to students through office hours and make responses within 24 hours
